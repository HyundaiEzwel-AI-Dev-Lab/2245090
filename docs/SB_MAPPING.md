# SB 매핑 기준

본 프로젝트는 SB 전체를 픽셀 단위로 복제하기보다, 실제 개발 가능한 핵심 업무 흐름을 우선 구현한다.

## 반영한 IA

| SB 메뉴 | 구현 위치 | 구현 수준 |
|---|---|---|
| 대시보드 | `/` | 프로젝트/지연 업무 요약 |
| 내업무 | `/my-work` | WBS 담당자 기준 카드형/캘린더형 |
| 프로젝트 등록/현황 | `/projects` | 등록, 검색, 상태/오픈월 필터 |
| 프로젝트 정보 | `/projects/{id}` | 상세, 요구사항, WBS, 테스트, 결함 통합 |
| 요구사항 관리 | `/projects/{id}` | 등록/확정 |
| WBS 관리 | `/projects/{id}` | 일정, 착수, 완료 |
| 단위/DEV/운영 테스트 | `/projects/{id}` | 테스트 단계 enum으로 관리 |
| 결함관리 | `/projects/{id}` | 테스트 실패 시 자동 생성 |
| 시스템 관리 | `/system/users` | POC 진입 화면 |

## 핵심 업무 흐름

1. 프로젝트 등록
2. 요구사항 등록
3. 요구사항 확정
4. WBS 작업단위 자동 생성
5. WBS 일정/착수/완료 처리
6. 내업무 반영
7. 테스트 수행
8. 실패 시 결함 자동 생성
9. 변경이력 적재

## 미구현/확장 영역

- 실제 로그인/SSO/인사 DB 연동
- 승인 결재선 상세
- 알림 발송 배치
- 테스트 라이브러리 일괄등록 상세
- 프로젝트별 탭 MDI 상세 UX
- 권한별 메뉴 접근 제어
